### 28.01.2026 — День 6
**Сделано:**
1. Изучил RAG (Retrieval Augmented Generation):
   - Создал тестовую базу знаний (8 документов о компании)
   - Настроил ChromaDB для хранения векторов
   - Использовал Sentence Transformers для embeddings
2. Выявил проблему: embedding модель слабая для русского языка
3. Реализовал гибридный поиск (BM25 + Vector):
   - BM25 — поиск по ключевым словам
   - Vector — семантический поиск
   - Hybrid — комбинация обоих методов
4. Интегрировал с DeepSeek для генерации ответов

**Технические заметки:**
- `paraphrase-multilingual-MiniLM-L12-v2` — бесплатная модель embeddings, но слабая для русского
- BM25 спасает когда vector search не работает (точное совпадение слов)
- ChromaDB хранит векторы в памяти, для продакшна нужен PersistentClient
- Гибридный поиск: `alpha * vector_score + (1-alpha) * bm25_score`

**Созданные материалы:**
- `notebooks/05_rag/rag_example.ipynb` — полный RAG пайплайн

**Выводы по RAG:**
- Для русского языка нужна лучшая модель: `intfloat/multilingual-e5-large` или OpenAI embeddings
- Гибридный поиск лучше чистого vector search
- Для продакшна: chunking больших документов, reranking, персистентность
